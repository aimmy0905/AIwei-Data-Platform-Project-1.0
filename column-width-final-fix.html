列宽问题最终修复！

CSS选择器问题已解决，表格布局现在正常显示

🔍 问题根因分析

经过深入分析发现，问题不在于列宽数值，而在于CSS选择器的匹配问题：

  • HTML模板中使用了 :class="selectedDepartment"
  • 这意味着实际的CSS类名是 operations 或 sales
  • 但CSS选择器写的是 .table-row.operations，需要同时匹配两个类
  • 导致样式无法正确应用到表格行

✅ 解决方案

添加了默认的表格行布局，确保所有表格行都有正确的列宽设置：

/* 确保默认表格行也有合适的布局 */ .table-row { grid-template-columns: 200px 100px 120px 120px 120px 120px 120px 100px 100px 120px; } /* 运营部门表格布局 */ .table-row.operations { grid-template-columns: 200px 100px 120px 120px 120px 120px 120px 100px 100px 120px; } /* 销售部门表格布局 */ .table-row.sales { grid-template-columns: 200px 100px 120px 120px 120px 120px 100px 100px 120px; }
